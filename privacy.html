Privacy Policy

Last updated: March 19, 2026

MailMapr does not collect, transmit, or share any personal data. All email analysis is performed entirely on your Mac. No data ever leaves your device.

1. Data Collection

MailMapr does not collect any personal information. Specifically:

  • No user accounts or registration required
  • No analytics or telemetry data collected
  • No usage tracking or behavioral profiling
  • No crash reports sent to external servers
  • No advertising identifiers used
  • No cookies or web tracking

2. Email Data Processing

MailMapr processes email files (EML, EMLX, MSG, MBOX) that you explicitly import into the application. This processing:

  • Occurs entirely on your local device
  • Is stored only in the application's sandboxed container on your Mac
  • Is never transmitted to any external server or cloud service
  • Can be deleted at any time by removing projects within the app

3. Network Usage

MailMapr includes a network entitlement solely to support report generation features (HTML rendering). The application:

  • Does not make any outbound network requests to external servers
  • Does not phone home or check for updates externally
  • Does not transmit any user data over the network

4. Data Storage

All application data is stored locally within the macOS Application Support directory, protected by the App Sandbox. This includes:

  • Imported email artifacts and their analysis results
  • Project configurations and trust lists
  • Generated forensic reports
  • Application preferences

You have full control over this data and can delete it at any time through the app or by removing the application container.

5. System Permissions

MailMapr requests access to the following system resources, all within the macOS App Sandbox:

  • Documents, Downloads & Desktop folders — For importing email files and exporting forensic reports. Access is granted only through user-initiated file dialogs or drag-and-drop.
  • User-selected files (read/write) — To open email files you choose and save exported reports.
  • Print — To print email content and reports.

MailMapr does not access any files without your explicit action.

6. Required Reason APIs

In compliance with Apple's App Store requirements, MailMapr declares the following Required Reason API usage:

  • File timestamp APIs — Used to display email file dates in the forensic analysis timeline. No data is sent externally.
  • UserDefaults — Used to store your application preferences (language, appearance, settings). Data remains on-device.

7. Third-Party Services

MailMapr does not integrate with any third-party services, SDKs, or analytics platforms. There are no external dependencies that process user data.

MailMapr uses two open-source libraries (ZIPFoundation and OLEKit) for file parsing only. These libraries operate entirely offline and do not transmit any data.

8. GDPR & CCPA Compliance

Because MailMapr does not collect, store, or transmit any personal data to external servers, the data protection obligations under GDPR (General Data Protection Regulation) and CCPA (California Consumer Privacy Act) are inherently satisfied:

  • No personal data is collected or processed by us
  • No data is shared with third parties
  • No cross-border data transfers occur
  • All data remains exclusively under your control on your local device

9. Children's Privacy

MailMapr is a professional forensic analysis tool designed for cybersecurity professionals, legal teams, and IT administrators. It does not target, market to, or knowingly collect information from children under 13 (or under 16 in the EU).

10. Data Retention & Deletion

Since all data is stored locally on your Mac:

  • You can delete individual projects and their data from within the app at any time
  • Uninstalling MailMapr removes the application and its sandboxed container
  • No residual data is retained on any external server, as none is ever transmitted

11. Changes to This Policy

If we update this privacy policy, the changes will be posted on this page with an updated revision date. Continued use of MailMapr after changes constitutes acceptance of the revised policy.

12. Contact

If you have questions about this privacy policy, please contact us at:
